# Zod `.optional()` не защищает от пустых строк в env-валидации

**Date:** 2026-04-23 00:00
**Files:** `apps/web/lib/env.ts`
**Severity:** high (fail-fast на старте dev/prod → приложение не поднимается)

## Problem

При первом локальном запуске `pnpm --filter web dev` после копирования `.env.example → apps/web/.env.local` (где большинство переменных оставлены как `KEY=` — пустые placeholder'ы) приложение падало на старте с ошибкой `Invalid server env` в `instrumentation.ts → getServerEnv()`:

```
- ANTHROPIC_API_KEY: String must contain at least 1 character(s)
- TELEGRAM_BOT_TOKEN: String must contain at least 1 character(s)
- TELEGRAM_WEBHOOK_SECRET: String must contain at least 1 character(s)
- ...
- UPSTASH_REDIS_REST_URL: Invalid url
- SENTRY_DSN: Invalid url
- SENTRY_AUTH_TOKEN: String must contain at least 1 character(s)
```

**Парадокс:** все переменные из списка (кроме `ANTHROPIC_API_KEY`) были явно помечены `.optional()` в Zod-схеме. Ожидалось, что их отсутствие молча пропустится. По факту — падение.

## Root Cause

Zod-методы `.optional()` и `.url()` / `.min(1)` имеют принципиально разную семантику «отсутствия»:

- `.optional()` означает **«принимает `undefined`»**, но НЕ пустую строку. `""` — это валидная строка, она проходит `.optional()` и уходит во внутренний валидатор.
- `.min(1)` падает на `""` с сообщением `must contain at least 1 character(s)`.
- `.url()` падает на `""` с сообщением `Invalid url`.

Когда `.env.local` содержит `TELEGRAM_BOT_TOKEN=` (пустое значение, не «переменная отсутствует»), `dotenv` / Next.js парсят это как `process.env.TELEGRAM_BOT_TOKEN = ""` — реальная пустая строка. Zod получает её и прогоняет через `z.string().min(1).optional()`:

1. Внешний `.optional()`: «это не `undefined` — идём дальше»
2. Внутренний `.min(1)`: «длина 0 < 1 — **fail**»

То же для `z.string().url().optional()` и пустой строки.

Дополнительный слой: `ANTHROPIC_API_KEY` был помечен как **required** (`z.string().min(1)`) — это отдельная, правомерная ошибка, но на Sprint 1 (baseline приложения, без AI) Claude ещё не используется. Required-флаг сработал технически правильно, но преждевременно по роадмапу.

**Мета-причина:** `.env.example` спроектирован как шаблон с пустыми placeholder'ами (чтобы не выглядеть как «установленный» конфиг — см. prevention из патча `2026-04-22-22.44.md`). При этом Zod-схема не была адаптирована под эту конвенцию. То есть шаблон и валидатор жили в разных ментальных моделях: шаблон говорит «пусто = не заполнено», валидатор говорит «пусто = невалидно».

## Solution

Введён хелпер `optionalEnvString` в `apps/web/lib/env.ts`, который делает preprocess `"" → undefined` **перед** применением inner-валидатора:

```typescript
function optionalEnvString(inner: z.ZodTypeAny = z.string().min(1)): z.ZodType {
  return z.preprocess((v) => (v === '' ? undefined : v), inner.optional());
}

const optionalUrl = () => optionalEnvString(z.string().url());
```

Все опциональные поля переведены на этот хелпер (`TELEGRAM_*`, `YOOKASSA_*`, `CRYPTOBOT_*`, `UPSTASH_*`, `TRIGGER_API_KEY`, `SENTRY_*`, `DATABASE_URL*`, `NEXT_PUBLIC_SENTRY_DSN`, `NEXT_PUBLIC_APP_URL`).

`ANTHROPIC_API_KEY` переведён в опциональные (Sprint 1.5+); `.default('claude-opus-4-6')` у `ANTHROPIC_MODEL` сохранён. Когда Telegram + AI milestone активируется, required-флаг можно будет вернуть на `TELEGRAM_BOT_TOKEN` и `ANTHROPIC_API_KEY` — **в тот же спринт**, где они реально становятся нужны.

Проверка: `pnpm --filter web dev` с пустыми placeholder-значениями в `.env.local` теперь поднимается за ~250ms без ошибок; `pnpm --filter web typecheck` и `build` зелёные.

## Prevention

1. **Никогда не использовать сырой `.optional()` на env-строках.** В проекте есть ровно один корректный способ — хелпер `optionalEnvString` / `optionalUrl`. Любой новый `z.string()...optional()` в `env.ts` — это потенциальный регресс.

2. **Сделать optional-по-умолчанию, required — осознанно.** Env-переменная должна становиться required только в тот момент, когда без неё приложение реально не может работать (`SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` — да; `TELEGRAM_BOT_TOKEN` — только в спринте, где подключается бот). Преждевременные required'ы ломают DX.

3. **Ментальная модель шаблона = ментальной модели валидатора.** `.env.example` с пустыми placeholder'ами — это контракт «пусто = не установлено». Валидатор обязан соблюдать этот контракт. Любое изменение одного требует ревью другого.

4. **Дженерик хелпер на preprocess '' → undefined — общеупотребимый паттерн для Zod + env.** Стоит добавить в `docs/coding-standards.md` раздел «Env-валидация» с правилом: использовать хелпер, не сырой `.optional()`.

5. **На будущем CI-скрипте schema↔template (патч `2026-04-22-22.44.md`):** также проверять, что каждая `optional`-переменная в Zod обёрнута через `optionalEnvString` (AST-проверка), а не через сырой `.optional()` — защита от копи-паста.

6. **Sprint transition checklist:** когда milestone из роадмапа активируется и соответствующая переменная становится обязательной — одновременно с этим коммитом нужно убрать `optionalEnvString`-обёртку с неё и добавить в README «переменная теперь обязательна». Иначе валидатор будет молча пропускать отсутствие critical-переменной.
